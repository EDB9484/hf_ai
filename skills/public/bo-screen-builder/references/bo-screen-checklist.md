# BO 화면 체크리스트

## 1) 요청 해석

- 도메인 패키지 이름을 확정했는가 (`admin.member` 같은 형태)
- 화면 유형이 목록/상세/등록 중 무엇인지 확인했는가
- 필수 검색 조건과 표시 컬럼을 정리했는가

## 2) 백엔드 구조

- Controller가 요청/응답 조합만 수행하는가
- Service가 필터링/변환/페이징 로직을 담당하는가
- 샘플 데이터 구조가 추후 MyBatis로 교체 가능하게 분리됐는가

## 3) 템플릿 구조

- 메뉴, 검색 폼, 결과 테이블, 페이징이 존재하는가
- `th:value`, `th:checked`, `th:selected`로 상태 유지가 되는가
- 일자/일시 검색 조건을 달력 입력(`date`/`datetime-local`)으로 제공하는가
- 일자/일시 검색 조건의 기본값이 당일로 설정되는가
- 일자 검색의 `전체` 버튼이 `2000-01-01 ~ 2999-12-31`로 동작하는가
- 검색 버튼/초기화 버튼이 가운데 정렬이며 `검색 -> 초기화` 순서로 배치되는가
- 검색 조건이 `label + input` 단위 그룹으로 묶여 있는가
- 기획서 기준의 행/그룹 구성을 유지하고 임의로 과도한 한 줄 압축을 하지 않았는가
- 라벨 폭이 고정되고 우측 정렬되어 input 시작선이 맞춰지는가
- 빈 결과 메시지가 조건부 렌더링되는가

## 4) 페이징 규칙

- 페이지당 노출 수 선택 UI가 결과 표 우측 상단에 위치하는가
- 첫 페이지에서 이전 버튼 처리 확인
- 마지막 페이지에서 다음 버튼 처리 확인
- 잘못된 페이지 번호 입력 시 방어 로직 확인

## 5) 완료 전 검증

- `./gradlew test` 실행 성공 여부 확인
- 테스트가 없다면 수동 확인 항목을 명시
- 기존 사용자 변경사항을 건드리지 않았는지 확인

## 기본 응답 템플릿

- 생성 파일 목록
- 검색/필터/페이징 동작 요약
- 검증 명령 실행 결과
- 남은 수동 점검 포인트
